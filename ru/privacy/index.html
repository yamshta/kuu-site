Политика конфиденциальности KUU

Последнее обновление: 21 июля 2026 г.

Коротко: То, что вы говорите, — ваше. Сама аудиозапись не отправляется за пределы вашего устройства. Транскрибация происходит на вашем устройстве. Для организации (классификации с помощью ИИ) используется внешний ИИ, но отправляется только транскрибированный текст. Он используется исключительно для организации и не сохраняется (в версии для iOS можно переключиться на организацию только на устройстве в разделе «На устройстве» в Настройках; версия для Android отправляет данные только внешнему ИИ — опции организации на устройстве нет). Данные хранятся на вашем устройстве, а в версии для iOS также в вашей личной базе данных iCloud (в версии для Android — только на устройстве). Разработчик не хранит ваши данные и не может просматривать полученную информацию. Вы можете в любой момент удалить все данные из приложения. Приложение выполняет только минимально необходимые сетевые запросы для обработки платежей (StoreKit на iOS / RevenueCat на Android) и показа рекламы Google AdMob, и эта информация никогда не включает ваши высказывания (рекламу можно отключить, оформив подписку KUU+). Для улучшения качества мы измеряем статистику использования, но эти данные также никогда не включают ваши высказывания (на iOS — с явного согласия пользователя; на Android по умолчанию отправляется статистика, не содержащая контента — подробности в Статье 14).

Статья 1 (Общие положения)

Приложение «KUU» (далее «Приложение») — это инструмент, который помогает вам выразить свои мысли вслух и упорядочить их. Существуют версии для iOS и Android, и настоящая Политика распространяется на обе. Приложение обрабатывает информацию только в минимальном объеме, необходимом для предоставления своих функций, ставя во главу угла защиту конфиденциальности пользователя.

Статья 2 (Собираемая и хранимая информация)

Приложение обрабатывает исключительно следующую информацию:

  1. Содержание ваших высказываний (аудиоданные) — Записанный звук сохраняется во временной области на устройстве только во время обработки транскрибации и немедленно удаляется после ее завершения. Аудиоданные не отправляются на сервер.
  2. Результаты транскрибации и организации (текст) — Сохраняются, чтобы вы могли их просматривать (в версии для iOS — на устройстве и в личной базе данных iCloud; в версии для Android — только на устройстве. Подробности в Статье 4).
  3. Настройки приложения — Значения, необходимые для работы приложения, такие как тема, размер шрифта, состояние «уровня воды в голове».

Приложение не собирает персональные данные, такие как имя, адрес электронной почты, номер телефона, данные о местоположении, контакты, календарь, фотографии или идентификаторы устройства.

Статья 3 (О распознавании речи и ИИ-классификации)

Распознавание речи (транскрибация) полностью выполняется на вашем устройстве iOS.

  • Распознавание речи: используется фреймворк Speech от Apple (на устройстве). Сама аудиозапись никогда не отправляется за пределы устройства.

ИИ-классификация (категоризация) использует внешний ИИ.

  • Отправляется только текстовое содержание (транскрибированный текст). Аудиозапись не отправляется.
  • Данные отправляются внешнему ИИ (Gemini от Google) через сервер разработчика (бэкенд).
  • Отправленный текст используется только для классификации и не сохраняется. Он также не используется для обучения ИИ.
  • В версии для iOS вы можете в любой момент переключиться на классификацию только на устройстве (FoundationModels от Apple / правила на устройстве) в настройках «На устройстве». В этом случае текст также не покидает пределов устройства.

О версии для Android: Версия для Android не предлагает способа организации (классификации), который выполняется исключительно на устройстве. При выполнении организации транскрибированный текст всегда отправляется внешнему ИИ (Gemini от Google через наш бэкенд). Переключателя «На устройстве», как в версии для iOS, нет. Отправляется только текстовое содержание; сама аудиозапись не отправляется, а отправленный текст используется исключительно для классификации, не сохраняется и не используется для обучения ИИ. Сама транскрибация (распознавание речи) выполняется на устройстве.

Статья 4 (Хранение и синхронизация)

Версия для iOS: Результаты транскрибации и организации хранятся только в вашей личной базе данных iCloud (CloudKit Private Database). Это хранилище, предоставляемое Apple, доступ к содержимому которого есть только у вас. Разработчик приложения не может просматривать или получать сохраненные данные. Использование iCloud регулируется политикой конфиденциальности Apple.

Версия для Android: Результаты транскрибации и организации хранятся только на этом устройстве. Автоматическая облачная синхронизация не выполняется. При смене устройства вы можете экспортировать данные в файл через «Голос и данные» в приложении и импортировать их на новом устройстве. Место сохранения файла выбирает сам пользователь (на устройстве, в вашем облачном хранилище и т. д.). Разработчик не имеет доступа к этому файлу.

Статья 5 (Цели использования)

Обрабатываемая информация используется исключительно в следующих целях:

  1. Создание транскрипции из голоса и ее отображение пользователю.
  2. Классификация транскрипций по категориям «Сейчас / Позже / Отложить / Отпустить» и их отображение пользователю.
  3. Хранение и отображение ранее произнесенных пользователем высказываний для их последующего просмотра.
  4. Сохранение настроек, необходимых для работы приложения.

Статья 6 (Использование внешних сервисов)

Для предоставления своих функций Приложение использует следующие внешние сервисы. Сама аудиозапись не отправляется ни одному из этих сервисов.

  • iCloud / CloudKit (только для iOS. Предоставляется Apple. Хранение и синхронизация только в вашей личной базе данных).
  • Распознавание речи (в версии для iOS — фреймворк Speech от Apple, в версии для Android — движок распознавания речи на устройстве. Оба выполняются на устройстве; аудиозапись не отправляется за его пределы).
  • Внешний ИИ (облачный) (ИИ-классификация. Отправляется только текстовое содержание. Используется исключительно для классификации, не сохраняется и не используется для обучения ИИ. В версии для iOS ее можно отключить в настройках «На устройстве», но версия для Android отправляет данные только внешнему ИИ. Подробности в Статье 3).
  • FoundationModels (только для iOS. Предоставляется Apple. Полностью выполняется на устройстве. Используется при включенной настройке «На устройстве» и как запасной вариант, когда внешний ИИ недоступен).
  • Платежные сервисы (в версии для iOS — Apple StoreKit, в версии для Android — RevenueCat. Управление покупкой, продлением, отменой и статусом подписки KUU+. Содержание высказываний не отправляется. О RevenueCat см. Статью 7 и Политику конфиденциальности RevenueCat).
  • Play Integrity API (через Firebase App Check. Только для Android) (Подтверждение целостности устройства и приложения, чтобы удостовериться, что запросы к API классификации отправляются из подлинного приложения. Не содержит ни содержания высказываний, ни информации, идентифицирующей пользователя).
  • Google AdMob (Google Mobile Ads SDK) (Только при отсутствии подписки KUU+: отображается один нативный рекламный блок между разделами на экране «Что вы сказали». Содержание высказываний не отправляется. Подробности в Статье 13).
  • Firebase Analytics (Предоставляется Google. Для улучшения качества приложения. В версии для iOS используется только в том случае, если пользователь явно дал согласие в Настройках; в версии для Android события об использовании, не содержащие контента, отправляются по умолчанию (в обоих случаях содержание высказываний не отправляется). В версии для iOS также используется Crashlytics при явном согласии, но версия для Android не содержит Crashlytics. Подробности в Статье 14).

Сервер Приложения выполняет только минимальную функцию ретрансляции запросов для ИИ-классификации и не хранит никакого контента (является stateless). Сервисы аутентификации, требующие личных аккаунтов, не используются.

Статья 7 (Передача данных третьим лицам)

Разработчик приложения не имеет доступа к содержанию ваших высказываний, результатам транскрибации или организации и не передает их третьим лицам.

В целях показа рекламы пользователям без подписки KUU+ в Google AdMob может отправляться информация, необходимая Google для показа рекламы, включая идентификаторы устройства, рекламный ID, язык и регион устройства, приблизительное местоположение и информацию о взаимодействии с рекламой (подробности в Статье 13; применяется политика конфиденциальности Google AdMob). При активной подписке KUU+ эта передача данных не происходит.

При оформлении подписки KUU+ в версии для Android информация о покупке (ID продукта, цена, дата покупки и т. д.) отправляется в RevenueCat, Inc. для обработки платежа и управления статусом подписки (активна/неактивна). Содержание высказываний не отправляется. Подробности об обработке данных RevenueCat см. в Политике конфиденциальности RevenueCat.

Раскрытие информации возможно только в случаях, предусмотренных законодательством, в соответствии с установленной процедурой.

Статья 8 (Удаление данных)

Пользователь может в любой момент удалить все данные через «Настройки → Голос и данные → Удалить сохраненные данные» в приложении. Это действие полностью удаляет данные на устройстве (а в версии для iOS — также и данные в личной базе данных iCloud). Восстановить удаленные данные невозможно.

При удалении Приложения данные на устройстве удаляются. Данные в iCloud для версии iOS можно удалить через Настройки Apple (Настройки → Apple ID → iCloud → Управлять хранилищем). В версии для Android данные хранятся только на устройстве, поэтому они удаляются вместе с приложением.

Статья 9 (Меры безопасности)

  • Версия для iOS: Временные аудиофайлы во время записи шифруются с помощью функции защиты файлов iOS (FileProtectionType.complete) и недоступны, когда устройство заблокировано. Связь с iCloud шифруется Apple с использованием SSL/TLS.
  • Версия для Android: Записанный звук не сохраняется на диск даже в качестве временного файла; он обрабатывается только в оперативной памяти и немедленно удаляется после распознавания. Сохраненные результаты транскрибации и организации хранятся в приватной области приложения, недоступной для других приложений, и исключены из автоматического облачного резервного копирования Android.
  • Все коммуникации с внешним ИИ шифруются (HTTPS/TLS). Сервер разработчика только ретранслирует запросы на классификацию и не хранит контент (является stateless).

Статья 10 (Использование несовершеннолетними)

Приложение имеет возрастной рейтинг 4+, но в силу своего характера (организация мыслей) предполагает использование лицами, умеющими читать и писать. Несовершеннолетним следует использовать приложение с согласия родителей или опекунов.

Статья 11 (Изменения в Политике конфиденциальности)

В настоящую Политику могут вноситься изменения в связи с изменениями в законодательстве, добавлением новых функций или изменениями в спецификациях фреймворков или политик платформ (Apple / Google). О существенных изменениях будет сообщено при обновлении приложения или на общедоступной странице этой Политики.

Статья 12 (Контактная информация)

По вопросам, касающимся настоящей Политики, обращайтесь через раздел «Разработчик» на странице приложения в App Store или Google Play, либо через «Настройки → Связаться с нами» в приложении.

Статья 13 (О рекламе и прозрачности отслеживания в приложениях)

В периоды, когда у вас нет подписки KUU+, Приложение отображает один нативный рекламный блок от Google AdMob на экране «Что вы сказали». Реклама отображается сдержанно, чтобы соответствовать стилю KUU.

  • Содержание ваших высказываний никогда не используется для рекламы (рекламные алгоритмы не анализируют ваши транскрипции, результаты классификации или темы).
  • Для показа рекламы Google AdMob может собирать идентификаторы устройства (включая IDFA), рекламный ID, приблизительное местоположение (Coarse Location), диагностические данные и данные о взаимодействии с продуктом (взаимодействие с рекламой в приложении).
  • Версия для iOS: Запрос App Tracking Transparency (ATT) отображается один раз, непосредственно перед показом первой рекламы. Реклама будет показываться даже в случае отказа, но информация, отправляемая в Google, будет ограничена (без персонализации). Вы можете изменить разрешение ATT в любое время в «Настройках» iOS → «Конфиденциальность и безопасность» → «Отслеживание».
  • Версия для Android: ATT — это механизм, эксклюзивный для iOS, и на Android он отсутствует. Вместо него для показа рекламы используется рекламный идентификатор Google (Advertising ID). Вы можете отказаться от персонализации рекламы или сбросить свой рекламный ID в «Настройках» устройства → «Конфиденциальность» → «Реклама» (формулировка может отличаться в зависимости от устройства и версии Android). Кроме того, версия для Android следует правилам управления согласием (UMP), отображаемым в соответствующих регионах, таких как ЕС.
  • Подписка на KUU+ полностью отключает рекламу и связанную с ней передачу данных.
  • Подробности об обработке данных Google AdMob см. в Политике конфиденциальности Google AdMob.

Статья 14 (Об использовании Firebase Analytics / Crashlytics)

Принцип явного согласия (opt-in), описанный в этой статье, применяется к версии для iOS. Информацию о версии для Android см. в разделе «О версии для Android» в конце этой статьи.

Версия для iOS может использовать Firebase Analytics (сбор статистики использования) и Firebase Crashlytics (отчеты о сбоях) от Google для улучшения качества приложения и немедленного выявления проблем в работе. Эта функция по умолчанию ВЫКЛЮЧЕНА (данные не отправляются) и активируется только в том случае, если пользователь явно дал согласие в «Настройках → Данные и диагностика».

  • Отправляемая информация:
    • Анонимный идентификатор установки, автоматически генерируемый Firebase (на основе IDFV; не является прямым персональным идентификатором).
    • Агрегированные сигналы о событиях в приложении (завершение сеанса записи, просмотр/конверсия экрана оплаты, завершение онбординга и т. п. Числовые значения отправляются в обобщенном, грубом виде).
    • Трассировка стека (symbolicated) при аварийном завершении работы приложения.
  • Неотправляемая информация: Содержание ваших высказываний (аудио), результаты транскрибации, текст результатов ИИ-классификации и установленные вами названия тем не могут быть отправлены на уровне типов (реализация API не позволяет передавать строковые значения в SDK аналитики).
  • Пока согласие не дано, никакая связь с Firebase, включая передачу вышеуказанной информации, не происходит.
  • Как прекратить отправку: Вы можете в любой момент выключить переключатель в «Настройках → Данные и диагностика». При выключении предыдущий идентификатор установки удаляется, а все несохраненные отчеты о сбоях, хранящиеся на устройстве, также стираются.
  • Получателем данных является Google LLC (США). Применяется Информация о конфиденциальности Firebase от Google.

О версии для Android: Версия для Android использует Firebase Analytics для отправки событий об использовании, не содержащих контента (обобщенные значения, такие как переходы между экранами и количество использований функций), а также анонимного App Instance ID, генерируемого Firebase, в целях улучшения продукта. В отличие от iOS, эта функция включена по умолчанию. Содержание ваших высказываний (аудио), транскрипции, текст результатов организации и названия тем не могут быть отправлены благодаря дизайну API SDK аналитики, который не позволяет передавать ему строковые значения. Версия для Android не содержит Crashlytics и не отправляет отчеты о сбоях. Получателем данных является Google LLC (США); применяется Информация о конфиденциальности Firebase от Google.
